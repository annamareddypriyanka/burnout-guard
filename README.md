# 🧠 Burnout Guard

## 🚀 Overview

Burnout Guard is a privacy-first system designed to **predict and prevent burnout** using simple daily behavioral inputs.
Unlike traditional mental health apps that only track mood, this system analyzes patterns, detects early warning signals, and provides actionable insights.

---

## 🎯 Problem Statement

Working professionals often experience stress and burnout but lack:

* A simple way to track their mental state
* Early detection of burnout risks
* Private and personalized insights

Most existing tools are **reactive**, not preventive.

---

## 💡 Solution

Burnout Guard transforms mental health tracking into a **proactive system** by:

* Collecting daily inputs (mood, sleep, stress)
* Calculating burnout risk
* Explaining causes of burnout
* Detecting patterns over time
* Providing early warnings and recommendations

---

## ✨ Key Features

### 🔹 Daily Check-in

* Mood (1–5)
* Sleep hours
* Work stress level

---

### 🔹 Burnout Score

* Calculates risk level:

  * 🟢 Low
  * 🟡 Medium
  * 🔴 High
* Provides **“Why this score”** explanation

---

### 🔹 Insights

* Compares current data with previous entries
* Detects changes in:

  * Sleep
  * Stress
  * Mood

---

### 🔹 Behavior Pattern Analysis

* Identifies trends across days
* Example:

  * “Stress is highest on Mondays”
  * “Sleep decreases on weekdays”

---

### 🔹 Burnout Timeline

* Displays last few entries
* Shows progression over time

---

### 🔹 Early Warning System

* Detects:

  * Increasing stress
  * Decreasing sleep
  * Rising burnout score
* Alerts users before critical burnout

---

### 🔹 Weekly Summary

* Average sleep
* Average stress
* Overall trend

---

### 🔹 AI Chat Support

* Responds to user emotions
* Provides contextual suggestions

---

### 🔹 User Authentication

* Secure login using Firebase Authentication
* Personalized tracking for each user

---

### 🔹 Data Storage

* Stored securely in Firestore
* Linked to individual user accounts

---

## 🔒 Privacy First

* User data is private and secure
* No unnecessary data sharing
* Designed with minimal data collection

---

## 🛠️ Tech Stack

* Frontend: React (Lovable-generated UI)
* Backend: Firebase Firestore
* Authentication: Firebase Auth
* Logic: Custom rule-based burnout scoring system

---

## ⚙️ How It Works

1. User logs in
2. Submits daily check-in
3. Data is stored in Firestore
4. System calculates burnout score
5. Insights and patterns are generated
6. Early warnings are triggered if needed

---

## 📊 Example Use Case

A user logs:

* Sleep decreasing over 3 days
* Stress increasing

➡ System detects pattern
➡ Triggers warning
➡ Suggests intervention
